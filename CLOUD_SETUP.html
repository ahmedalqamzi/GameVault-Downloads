Optional cloud

Your library, on your cloud.

GameVault works without a server. Use this setup only when you want an off-device backup, autosync, or the same collection on several of your devices.

What you need

  • A Windows, macOS, or Linux computer
  • Node.js 22 or newer
  • A free Cloudflare account
  • About 10 minutes
  • Optional Twitch/IGDB credentials for live game metadata
Your app does not wait for this.

Install GameVault and use phone-only mode immediately. You can connect cloud later without reinstalling.

Guided setup

  1. Download and extract the setup kit on a computer.
  2. Install Node.js 22+ and create a Cloudflare account.
  3. On macOS/Linux run chmod +x setup-cloud.sh, then ./setup-cloud.sh.
  4. On Windows PowerShell run Set-ExecutionPolicy -Scope Process Bypass, then ./setup-cloud.ps1.
  5. Follow the prompts. Cloudflare login opens in your browser.
  6. At the end, copy the printed Worker URL and personal token into GameVault → Settings → Optional Cloud.
  7. Choose Save connection, then Sync.

What the helper does

It installs Cloudflare Wrangler, creates your D1 database, generates a random 64-character token, stores that token as a Worker secret, applies the complete GameVault schema, and deploys the API. It can also prompt securely for optional IGDB credentials.

The kit contains placeholders only. It does not contain another person's account, token, database ID, private game library, or metadata credentials.

Add game metadata

Live search, refreshed artwork, dates, scores, and trailers use IGDB. Create Twitch application credentials using the official IGDB authentication guide. The setup helper can store them as Worker secrets without putting them in source files.

Keep the token private

This API is intentionally single-owner. Anyone with its token can read and change the whole cloud library. Each friend should deploy a separate kit. Never share a URL/token pair or commit the generated .gamevault-sync-token file.
